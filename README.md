# Lexora

Lexora 是一个面向 iOS 的轻量化背单词应用构想。

它的目标不是做一个复杂的语言学习平台，而是提供一个干净、离线、可控的个人词库工具。用户可以导入自己的单词表，并通过类似卡片式的学习界面进行记忆和复习。

当前项目仍处于早期构思阶段，暂不包含实际代码实现。

## 核心定位

Lexora 的核心方向是：

* 仅支持 iOS
* 纯离线使用
* 支持从 CSV / TXT 导入单词表
* 提供轻量化的背单词界面
* 适配 Liquid Glass
* 不做账号系统、不做社区、不做复杂内容平台

## 设计目标

Lexora 希望保持简单、克制、直接。

相比大型背单词应用，Lexora 更关注以下几点：

* 用户拥有自己的词表
* 数据本地保存
* 导入流程清晰可靠
* 背词界面轻量、低干扰
* 复习过程不被广告、社交、推荐流打断
* 界面符合 iOS 原生体验

## 初步功能设想

### 词书管理

* 创建本地词书
* 查看词书列表
* 查看词书内单词
* 删除或重命名词书

### 单词导入

* 支持 CSV 导入
* 支持 TXT 导入
* 导入前预览内容
* 显示成功导入数量
* 显示格式错误行
* 部分行错误时，不影响其他单词导入

### 背词界面

背单词界面大致采用卡片式学习流程。

初步设想包括：

* 当前单词
* 图片或视觉辅助区域
* 释义展示
* 例句展示
* 熟悉 / 不熟悉标记
* 下一个单词
* 简洁的进度提示

整体交互可以参考常见图文卡片式背词应用，但 Lexora 会保持更轻量、更本地化、更少干扰。

### 复习系统

第一阶段只实现基础复习逻辑：

* 标记熟悉
* 标记不熟悉
* 查看不熟词
* 重新复习不熟词

复杂的间隔重复算法可以后续再考虑。

## 初步导入格式

### CSV

最小格式：

```csv
word,meaning
abandon,放弃
brief,简短的
```

扩展格式：

```csv
word,meaning,phonetic,example,tags
abandon,放弃,/əˈbændən/,He abandoned the plan.,cet4;verb
```

### TXT

一种可能的 TXT 格式：

```txt
abandon - 放弃
brief - 简短的
conduct - 进行；行为
```

具体格式仍待进一步确定。

## 平台方向

Lexora 只考虑 iOS 端。

后续技术方向暂定为：

* Swift
* SwiftUI
* 本地数据存储
* 原生 iOS 文件导入
* Liquid Glass 风格适配

具体技术选型会在真正开始开发前再决定。

## 暂不考虑

为了保持项目简单，暂不考虑：

* Android 版本
* Web 版本
* 账号登录
* 云同步
* 社区词库
* 排行榜
* 广告系统
* 在线词典
* AI 自动生成释义
* 订阅制商业化

这些功能并非永远不做，只是不属于第一阶段目标。
